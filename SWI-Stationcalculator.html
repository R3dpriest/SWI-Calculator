Q: Does effeciency work in the calculations?
A: yes set 'Adjust for max effenciency?' to 'Yes'
Q: What is this downscaling?
A: Long story short, the calculator works its way down from most difficult to simplest resource and suggests modules. If you toggle this to 'Downscale' it is allowed to reduce your represet values.
Q: Why select a module preference? Wont this just clog the UI?
A: Yes, it does, but with building materials taken into concideration i noticed significant resource differences for building. So if i want to keep those accurate i have to accomodate for it.
Q: How accurate is the in & export?
A: Relatively accurate... as long as i dont change anything. It reads all the fields converts that into a string and encodes it for shortening this makes it vunerable for change.
Q: I have a suggestion.
A: You can tag me in discord or find the tread, but please note I am not a proffesional programmer. I work with the tools I know.
Q: Will you add suggestion.
A: I have added a fair bith of suggestions already. Infact v2 is basically a suggestion of people saying 'We like this UI'. And the spreadsheet is rebuild under the same guise.
Q: How did you build this website?
A: I currently do not have a payed host. So this kept me from using backend calculations and data storage. Everything has been build with .css .js. and .html. This has the added bonus that you can see how i structure the data and suggest new modules or resources to me.
Q: I want to give you some data! Where do i find it and how do i structure it?
A-1: You can if you want to go through the hassle, unpack the mod files. Or you can check the ingame encyclopedia.
A-2: As for how i set up my data? Check out on this page. Once you see how modules and resources are tied together it become easy.
